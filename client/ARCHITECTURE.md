## 组件说明

### 1. 服务端组件

#### 1.1 客户端管理 API (`backend/api/client_monitor.py`)

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/client/register` | POST | 客户端注册 |
| `/api/client/heartbeat` | POST | 客户端心跳 |
| `/api/client/status` | GET | 获取所有客户端状态 |
| `/api/client/upload` | POST | 客户端文件上传 |
| `/api/client/disconnect` | POST | 客户端断开 |
| `/api/client/ws/{client_id}` | WebSocket | 客户端WebSocket连接 |

#### 1.2 客户端管理器 (`ClientManager`)

核心功能：
- **客户端注册**：为新客户端分配ID，记录连接信息
- **心跳管理**：监控客户端在线状态（5分钟无心跳视为离线）
- **统计追踪**：记录每个客户端的上传数量和异常检测结果
- **WebSocket管理**：维护客户端WebSocket连接

#### 1.3 前端客户端监控页面

位于 `实时监控` 标签后的 `🖥️ 客户端监控` 页面，显示：
- 在线/离线客户端数量统计
- 总上传文件数
- 每个客户端的详细信息卡片
- 客户端活动日志
- 客户端上传文件的检测结果

### 2. 客户端组件

#### 2.1 文件监控 (`watchdog`)

- 使用 `watchdog.observers.Observer` 监控指定目录
- 监听 `FileCreatedEvent` 事件检测新文件
- 文件稳定检测（等待文件写入完成）

#### 2.2 网络通信

**HTTP上传**：
- 使用 `httpx.AsyncClient` 进行异步HTTP请求
- 支持文件分块上传
- 超时配置：300秒

**WebSocket连接**：
- 使用 `websockets` 库
- 自动重连机制（最多3次）
- 心跳保持（每30秒）

#### 2.3 错误处理与重试

- **上传重试**：失败后自动重试，最多3次
- **指数退避**：重试间隔递增（5秒 × 重试次数）
- **断线重连**：WebSocket断开后自动重连
- **异常捕获**：全面的try-except包裹

```

        ▼
```

## 容错机制

### 1. 网络异常处理

| 异常类型 | 处理策略 |
|---------|---------|
| 连接超时 | 指数退避重试 |
| 连接断开 | WebSocket自动重连 |
| 服务器无响应 | 标记离线，定期重试连接 |
| 上传中断 | 重试3次后放弃，记录失败 |

### 2. 客户端状态恢复

- **重启恢复**：客户端重启后使用相同ID重新注册
- **任务恢复**：WebSocket重连后继续接收未完成任务的进度
- **文件续传**：检测到已处理文件不重复上传（基于文件名）

## 性能考虑

### 1. 并发控制

- **客户端端**：串行上传（避免网络拥塞）
- **服务端端**：
  - 任务队列管理
  - 单GPU串行执行（避免显存溢出）
  - 支持多GPU选择

### 2. 资源优化

- **文件去重**：相同文件名不重复处理
- **连接复用**：HTTP Keep-Alive + WebSocket长连接
- **增量更新**：只传输新检测到的文件

## 安全考虑

1. **客户端认证**：基于client_id的简单认证
2. **文件类型限制**：只允许特定音频格式
3. **IP白名单**：建议部署时配置防火墙
4. **HTTPS支持**：公网部署建议使用HTTPS/WSS
