## 1. 项目概述

### 1.1 项目背景

在扬声器(SPK)等音频设备的生产制造过程中，产品质量检测是确保产品合格的关键环节。传统的音频质检依赖人工听音，存在主观性强、效率低、一致性差等问题。随着深度学习技术的发展，基于AI的音频异常检测成为工业质检的重要方向。

### 1.2 核心价值

本项目构建了一套完整的**音频异常检测系统**，具有以下核心价值：

| 价值维度 | 具体体现 |
|---------|---------|
| **技术先进** | 集成30+种SOTA异常检测算法，涵盖重建、特征嵌入、流模型、知识蒸馏等多种技术路线 |
| **定位精准** | 独创的长音频目标定位技术，支持在任意长度音频中精准定位10秒检测片段 |
| **架构现代** | 采用前后端分离架构，支持分布式部署和多客户端并发 |
| **易用性强** | 提供Web可视化界面，支持离线检测、实时监控、特征聚类等多种工作模式 |
| **扩展性好** | 统一算法接口层，新算法接入成本低 |

## 2. 长音频目标音频定位技术

### 2.1 技术原理与实现方案

#### 2.1.1 问题定义

在实际生产场景中，待检测音频往往是长音频（数分钟到数十分钟），而深度学习模型需要固定长度的输入（如10秒）。因此需要在长音频中**精准定位**目标检测片段。

#### 2.1.2 三种定位方案对比

| 方案 | 原理 | 精度 | 速度 | 适用场景 |
|-----|------|-----|------|---------|
| **MFCC+DTW** | MFCC特征提取 + 动态时间规整 | 高(±0.1s) | 中 | 单参考音频，生产环境推荐 |
| **Shazam指纹** | 音频指纹哈希 + 时间对齐投票 | 高(±0.064s) | 快 | 多参考音频，复杂场景 |
| **互相关** | 时域信号互相关计算 | 中(±0.5s) | 快 | 快速原型验证 |

**核心代码逻辑：**

```python
class MFCCLocate:
    def extract_mfcc(self, audio_path, sr=16000, n_mfcc=13):
        """提取MFCC特征"""
        y, sr = librosa.load(audio_path, sr=sr)
        mfcc = librosa.feature.mfcc(y=y, sr=sr, n_mfcc=n_mfcc)
        return mfcc.T  # 转置为(时间帧, 特征维度)

    def dtw_match(self, ref_mfcc, query_mfcc):
        """DTW动态时间规整匹配"""
        # 使用余弦距离作为度量
        alignment = dtw(ref_mfcc, query_mfcc, dist_method='cosine')
        return alignment.index1, alignment.index2  # 返回对齐路径
```

### 2.2 算法流程与关键技术点

#### 2.2.1 PreciseSegmentLocator精确片段定位器

针对传统滑动窗口方式的性能瓶颈，我们开发了`PreciseSegmentLocator`：

```

│           PreciseSegmentLocator 全局指纹提取策略                 │
├─────────────────────────────────────────────────────────────────┤

│  长音频 (60s)                                                   │
│  ┌──────────────────────────────────────────────────────┐      │
│  │  1. 全局指纹提取 (一次)                               │      │
│  │     - 提取完整频谱图                                  │      │
│  │     - 检测全局峰值点                                  │      │
│  │     - 生成完整hash列表 (~3000个)                     │      │
│  └──────────────────────────────────────────────────────┘      │
│                         ↓                                      │

│  │  2. 批量数据库查询 (一次)                             │      │
│  │     - 批量查询所有hash                               │      │
│  │     - 获取匹配结果                                   │      │

│  │     - 找出所有超过阈值的匹配点                        │      │
```

#### 2.2.2 关键性能优化

| 优化点 | 传统方案 | 优化方案 | 效果 |
|-------|---------|---------|------|
| 指纹提取 | 滑动窗口(多次) | 全局提取(一次) | 减少重复计算 |
| 数据库查询 | N次查询 | 批量查询(1次) | 减少IO开销 |
| 定位精度 | 窗口级别(±5s) | Hash级别(±0.064s) | 精度提升80倍 |
| 多片段检测 | 不支持 | 支持 | 功能增强 |

#### 2.3.2 负偏移处理（重要）

Shazam算法中需要特别注意负偏移的处理：

```python
# 当 locate 返回负偏移时：
# offset_diff = -129 表示参考音频从查询音频的 8.26s 处开始
# 实际起始时间 = -offset_diff * hop_length / sr = 8.26s

# 正确处理方式：
if location.start_time < 0:
    # 表示参考音频在查询音频的 |offset| 秒处开始
    start_time = -location.start_time  # 取绝对值
    
# 错误处理（会导致切分位置错误）：
if start_time < 0:
    start_time = 0.0  # ❌ 这会丢失定位信息
```

### 3.3 前后端通信机制与安全策略

#### 3.3.1 WebSocket实时通信

```python
# WebSocket连接管理器
class WebSocketManager:
    def __init__(self):
        self.connections: Dict[str, WebSocket] = {}
    
    async def handle_connection(self, websocket: WebSocket, task_id: str):
        await websocket.accept()
        self.connections[task_id] = websocket
        try:
            while True:
                # 保持连接，接收心跳
                data = await websocket.receive_text()
        except:
            pass
        finally:
            del self.connections[task_id]
    
    async def send_progress(self, task_id: str, message: str, progress: float):
        if task_id in self.connections:
            await self.connections[task_id].send_json({
                "type": "progress",
                "message": message,
                "progress": progress
            })
```

#### 4.2.3 缓存策略

```python
# 设备列表缓存（避免频繁查询GPU）
_devices_cache = {
    "data": None,
    "timestamp": 0,
    "ttl": 10  # 10秒缓存
}

def _get_cached_devices():
    current_time = time.time()
    if _devices_cache["data"] is not None and \
       (current_time - _devices_cache["timestamp"]) < _devices_cache["ttl"]:
        return _devices_cache["data"]
    return None

# 算法实例缓存（避免重复加载模型）
_algorithm_cache = {}

def get_algorithm(name):
    if name not in _algorithm_cache:
        _algorithm_cache[name] = load_algorithm(name)
    return _algorithm_cache[name]
```

#### 5.3.1 异步任务队列

```python
# TaskManager 核心实现
class TaskManager:
    def __init__(self):
        self.task_queue = asyncio.Queue()
        self.tasks = {}  # task_id -> TaskInfo
        self.workers = []
        
    async def create_task(self, files, algorithm, device, **kwargs):
        task_id = str(uuid.uuid4())
        self.tasks[task_id] = {
            "id": task_id,
            "status": "queued",
            "progress": 0,
            "files": files,
            "algorithm": algorithm,
            "device": device,
            "created_at": datetime.now()
        }
        await self.task_queue.put(task_id)
        return task_id
    
    async def worker(self):
        while True:
            task_id = await self.task_queue.get()
            await self.process_task(task_id)
            self.task_queue.task_done()
    
    async def process_task(self, task_id):
        task = self.tasks[task_id]
        task["status"] = "processing"
        
        # 1. 音频预处理
        await self.send_progress(task_id, "音频预处理...", 10)
        
        # 2. 定位与切分
        await self.send_progress(task_id, "定位目标片段...", 30)
        
        # 3. 异常检测
        await self.send_progress(task_id, "异常检测推理...", 60)
        
        # 4. 结果汇总
        await self.send_progress(task_id, "汇总结果...", 90)
        
        task["status"] = "completed"
        task["progress"] = 100
        await self.send_progress(task_id, "检测完成", 100)
```

#### 5.3.2 WebSocket实时通信

```javascript
// 前端 WebSocket 客户端
class WebSocketClient {
    constructor(taskId) {
        this.taskId = taskId;
        this.ws = null;
        this.reconnectAttempts = 0;
        this.maxReconnectAttempts = 5;
    }
    
    connect() {
        this.ws = new WebSocket(`ws://server:8004/ws/progress/${this.taskId}`);
        
        this.ws.onmessage = (event) => {
            const data = JSON.parse(event.data);
            this.handleMessage(data);
        };
        
        this.ws.onclose = () => {
            if (this.reconnectAttempts < this.maxReconnectAttempts) {
                setTimeout(() => this.connect(), 3000);
                this.reconnectAttempts++;
            }
        };
    }
    
    handleMessage(data) {
        switch(data.type) {
            case 'progress':
                updateProgress(data.progress, data.message);
                break;
            case 'result':
                displayResult(data.result);
                break;
            case 'error':
                showError(data.message);
                break;
        }
    }
}
```
