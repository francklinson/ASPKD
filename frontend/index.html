什么是特征聚类？

特征聚类分析通过提取音频的特征向量，使用降维和聚类算法将相似的音频分组，并识别出异常样本。


特征提取器说明：

  • MFCC (Mel-frequency cepstral coefficients)：传统声学特征，计算速度快，适合快速预览和语音分析
  • Mel Spectrogram：梅尔频谱图特征，反映人耳听觉特性，适合音乐分类和声音事件检测
  • HuBERT：Facebook自监督语音表示模型，在语音识别任务上表现优异，适合语音聚类
  • WavLM：微软自监督语音模型，具备语音内容和说话人信息建模能力，适合声纹识别
  • XLSR-Wav2Vec2：跨语言语音表示模型，支持多语言音频处理，适合多语种场景
  • AST (Audio Spectrogram Transformer)：音频Transformer模型，基于ImageNet预训练，适合复杂声学场景
  • MERT：音乐理解Transformer，针对音乐数据优化的模型，采样率24kHz，适合音乐分析

使用建议：

  • 至少上传2个音频文件
  • 建议上传10-50个文件以获得更好的聚类效果
  • 音频时长建议在3-10秒之间
  • 同类音频应该有相似的特征

什么是零样本检测？

基于 MuSc (ICLR 2024) 算法的零样本异常检测，无需任何训练数据或标注，直接通过测试图像间的相互关系来检测异常。


骨干网络说明：

  • CLIP ViT-B/32：小模型，推理速度快，适合快速预览
  • CLIP ViT-B/16：中等模型，速度和精度均衡
  • CLIP ViT-L/14：大模型，精度高，推荐使用
  • DINOv2 ViT-B：轻量级，适合边缘设备部署
  • DINOv2 ViT-L：高精度，适合对准确性要求高的场景

参数说明：

  • 异常阈值：判定异常的分数阈值（0-1），分数高于此值判定为异常。检测完成后可通过滑动条实时调整，结果会立即刷新
  • 聚合度列表：LNAMD模块的局部聚合度，默认1,3,5表示多尺度分析
  • 批处理大小：根据显存调整，显存不足时减小此值

使用建议：

  • 至少上传5个同类别图像文件
  • 建议上传10-20个文件以获得更好的检测效果
  • 图像应属于同一类别（如同一类工业产品）
  • 正常样本应占多数，异常样本会被识别出来
  • 图像尺寸建议 336x336 或 518x518

什么是少样本检测？

基于 SubspaceAD (CVPR 2026) 算法的少样本异常检测，仅需 1-8 张正常样本作为参考，即可检测新的异常图像。无需训练，直接通过 PCA 子空间建模来识别异常。


骨干网络说明：

  • DINOv2-Large@672px：最高精度，推荐使用
  • DINOv2-Large@518px：高精度，速度与精度均衡
  • DINOv2-Large@336px：均衡配置，适合大多数场景
  • DINOv2-Base：轻量级，适合边缘设备
  • DINOv2-Small：快速推理，适合实时应用

参数说明：

  • 参考样本数 (k-shot)：用于建模的正常样本数量，1-shot 即可工作，更多样本可提高稳定性
  • PCA 解释方差比例：保留的主成分比例，0.99 是推荐值
  • 评分方法：重构误差是默认方法，也可选择马氏距离等
  • 异常阈值：判定异常的分数阈值，检测完成后可实时调整

使用建议：

  • 参考样本必须是正常（无异常）的图像
  • 参考样本应覆盖正常情况的变化范围
  • 1-shot 即可工作，建议 4-shot 以获得更好效果
  • 测试图像可以与参考样本同类或不同类
  • 图像尺寸建议 518x518 或 672x672

什么是参考音频？

参考音频是用于 Shazam 音频指纹识别的标准音频片段。系统会将这些音频的特征（指纹）存储在数据库中，用于后续的音频识别和定位。


使用建议：

  • 建议音频时长在 5-30 秒之间
  • 选择音频中特征明显的片段
  • 避免使用过于相似的不同音频
  • 为音频设置易于识别的名称
