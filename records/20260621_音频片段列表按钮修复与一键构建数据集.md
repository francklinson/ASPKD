## 任务 1：修复音频片段列表控件按钮不生效

### 问题表现

预览数据集构建会话后，音频片段列表内的「分类标签」和「筛选标签」按钮点击无任何视觉效果。

### 根因

`loadSegments()` 函数在第 2035 行存在守卫判断：

```javascript
if (currentV2SessionId) {
    logMessage('当前在会话浏览模式', 'info');
    return;
}
```

当用户预览一个 V2 会话后，`currentV2SessionId` 被设置。此后任何按钮点击（分类/筛选标签）都会触发 `loadSegments()`，但该函数立即 return，不执行任何过滤和渲染逻辑。用户看到按钮被点击但列表纹丝不动。

### 修改方案

**5 处改动**，全部在 `frontend/dataset.html`：

1. **`loadSegments()`** — 不再直接 return，改为调用 `filterV2Segments()` 进行本地过滤
2. **新增 `filterV2Segments()`** — 对会话内的 `currentSegments` 按 `currentCategory`/`currentFilter` 过滤，重新渲染，处理选中片段消失的边界
3. **新增 `getActiveSegments()`** — 返回当前生效的片段列表（会话模式下为过滤后的列表），供导航按钮统一使用
4. **`updateNavButtons()`/`goToPrevious()`/`goToNext()`** — 全部改用 `getActiveSegments()`，确保上一个/下一个在过滤后的列表内导航
5. **`previewV2Session()`** — 预览新会话时重置 `currentCategory`/`currentFilter` 为 "all"，并同步 UI 标签高亮

**额外修正**：
- 将 `currentV2SessionId` 声明从底部（line 3877）移到顶部全局状态区，消除重复声明
- 标注完成后（line 4053）改为调用 `filterV2Segments()` 而非 `renderV2SegmentsList(currentSegments)`，保持过滤状态

## 任务 2：一键构建数据集（跨会话重构）

### 背景

原有的「确认导入正式数据集」按钮绑定到单个会话：
- 必须先预览一个会话，按钮才可见（`display:none` → 仅在 `previewV2Session()` 中显示）
- 只导入当前会话内的片段
- 多会话需重复操作

用户要求改为：每个样本标注结果已实时保存，一键将所有会话中已标注的样本（含之前标注的）统一划分训练集/测试集。

### 后端改动

文件：`backend/api/dataset_builder_v2.py`（+252/-78）

**提取公共函数 `_import_single_segment()`**（line 904）

将原 `confirm_to_dataset()` 中逐片段的拷贝逻辑提取为独立函数，供新旧两套导入流程复用：

```python
def _import_single_segment(segment, train_ratio=10, target_category=None):
    """拷贝一个片段到 data/spk/{category}/{split}/{label}/"""
    # 确定类别 → 标签 → train/test划分 → 拷贝文件 → 更新状态
    return (success, {"category", "split_type", "label"})
```

**新增 `confirm_all_annotated_to_dataset()` 函数**（line 1064）

```python
async def confirm_all_annotated_to_dataset(reference_audios, train_ratio):
    """遍历 BUILD_SESSIONS，收集所有已标注片段，调用 _import_single_segment()"""
```

逻辑：
1. 遍历 `BUILD_SESSIONS` 所有会话
2. 收集 `manual_label IS NOT NULL AND status == "annotated"` 的片段
3. 按 `reference_audios` 过滤（可选）
4. 对每个片段调用 `_import_single_segment()`
5. 保存所有受影响会话（持久化 status → `in_dataset`）
6. 返回导入数/失败数/各参考音频的划分明细

**新增请求模型**：

```python
class ConfirmAllAnnotatedRequest(BaseModel):
    reference_audios: Optional[List[str]] = None
    train_ratio: int = 10
```

**新增端点** `POST /api/dataset/confirm-all-annotated` — 无需 session_id，纯全局操作。

**重写 `confirm_to_dataset()`** — 循环体改为调用 `_import_single_segment()`，保留原功能不变。

文件：`frontend/dataset.html`（+186/-24）

**新增 JS 函数 `buildDatasetFromAllAnnotated()`** — 流程：
1. 一个 prompt 询问训练:测试比例（默认 10）
2. 可选 prompt 选择参考音频（仅当有多个参考音频时）
3. POST `/api/dataset/confirm-all-annotated` → 显示结果
4. 刷新统计卡片和会话列表

**`renderV2SegmentsList()` 增强** — 识别 `seg.status === 'in_dataset'` 时显示蓝色 "已导入" 标签。

**`previewV2Session()` 增强** — 映射片段时增加 `status` 字段传递。

### 边界处理

| 场景 | 行为 |
|------|------|
| 没有任意会话有已标注片段 | 返回失败 + 提示信息 |
| 部分会话有标注、部分无 | 只导入有标注的 |
| 某片段已导入过（status=in_dataset） | 自动跳过 |
| 只有预测标签、无人工作标注 | 不导入（仅 manual_label 非空才算） |
| 源音频文件已丢失 | copy2 抛异常 → 单个失败计数 |
| 重复点击按钮 | 幂等：第二次无 annotated 片段，early return |
| 用户取消 prompt | 什么都不做 |
