## 一、mamba_ssm / causal-conv1d 安装

### 环境信息
- Python 3.12 | PyTorch 2.8.0+cu128 | CUDA 12.8 | CXX11_ABI=True | Linux x86_64

### 安装过程

1. **pip 在线安装失败**：causal-conv1d build dependencies 问题，mamba_ssm 依赖会拉升 torch 版本（2.7→2.13）
2. **离线 whl 方案**：从 GitHub Releases 下载预编译 whl
   - 文件名格式：`{pkg}+cu12torch{X.Y}cxx11abi{TRUE|FALSE}-cp312-cp312-linux_x86_64.whl`
   - **关键**：whl 的 torch 版本必须与环境中实际 torch 版本严格匹配，否则 CUDA 扩展 `undefined symbol`
3. **版本匹配**：
   - 第一次下载了 `torch2.7` 版本的 whl，在 torch 2.8.0 下报 `undefined symbol: _ZN3c104cuda9SetDeviceEa`
   - 重新下载 `torch2.8` 版本后正常

### 安装命令
```bash
# .venv 环境
.venv/bin/pip install causal_conv1d-1.6.2.post1+cu12torch2.8cxx11abiTRUE-cp312-cp312-linux_x86_64.whl
.venv/bin/pip install mamba_ssm-2.3.2.post1+cu12torch2.8cxx11abiTRUE-cp312-cp312-linux_x86_64.whl --no-deps
```

### CXX11 ABI 说明
- GCC 5.0 引入的 C++ 标准库 ABI 变更（`std::string` SSO vs COW）
- PyTorch 编译时选定 ABI，所有 C++ 扩展必须匹配
- 查询：`torch._C._GLIBCXX_USE_CXX11_ABI` → True/False
- **不匹配时**：`ImportError: undefined symbol` 或运行时崩溃

### torch 版本升级事件
- mamba_ssm `--no-deps` 未用时会自动拉 torch 2.13.0，破坏 torchvision/torchaudio 兼容性
- 用户决定将 torch 统一升级到 2.8.0+cu128，并重装配套的 torchvision/torchaudio

### 最终安装版本
| 包 | 版本 |
|---|---|
| torch | 2.8.0+cu128 |
| torchvision | 0.23.0+cu128 |
| torchaudio | 2.8.0+cu128 |
| causal-conv1d | 1.6.2.post1 |
| mamba_ssm | 2.3.2.post1 |

## 二、代码修复

### 1. mamba_ssm 可选导入（`algorithms/ADer/model/mambaad.py`）
```python
try:
    from mamba_ssm.ops.selective_scan_interface import selective_scan_fn, selective_scan_ref
except ImportError:
    selective_scan_fn = None
    selective_scan_ref = None
```
- 未安装 mamba_ssm 时，MambaAD 以 assigner-only 模式运行，不阻止其他模型加载

### 2. ViTAD 预训练权重加载修复（`algorithms/ADer/model/vitad.py`）
- 原问题：裸 `except` + 无限循环，离线环境下卡死
- 修复：有限重试（10次）+ 离线降级（`pretrained=False`）

### 3. RealNet 空 checkpoint_path 修复（`algorithms/ADer/model/realnet.py`）
- 原问题：空字符串路径传入 `torch.load()` 报 `FileNotFoundError`
- 修复：检查路径有效性，无效时通过 timm `pretrained=True` 在线下载 fallback

### 4. WinClip 离线权重加载（`algorithms/anomalib/models/image/winclip/torch_model.py`）
- 定义本地权重路径常量 `_LOCAL_WEIGHTS`
- 优先使用本地 `.pt` 文件，离线模式无需下载

### 5. accimage 可选导入（`algorithms/ADer/util/vis.py`, `data.py`）
- accimage 不可通过 pip 安装，改为 try/except 可选导入

### 6. RDpp 配置目录兼容
- 目录名 `rd++` 中的 `+` 不能作为 Python 模块名
- 创建 `rdpp_real/` 目录并建立符号链接 `rdpp -> rdpp_real`

### 7. HF 缓存路径修正（`algorithms/ader_adapter.py`）
- `HUGGINGFACE_HUB_CACHE` 路径少了一层 `hub`，修正为 `models/pre_trained/huggingface/hub`

### 8. imgaug + NumPy 2.0 兼容性修复
- `np.sctypes` 在 NumPy 2.0 中被移除
- 修改 `.venv/lib/python3.12/site-packages/imgaug/imgaug.py`，添加 try/except 回退
- 系统 Python 和 .venv 的 imgaug 都已修复
