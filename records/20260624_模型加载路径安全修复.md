# 模型加载路径安全问题修复

> 修复时间：2026-06-24
> 对应分析：`records/20260624_213839_模型文件与模块映射分析.md`

## 背景

审计发现部分模型加载路径使用相对路径（依赖 CWD 为项目根目录），以及个别 `~/.cache/clip` 系统路径 fallback，在 CWD 非项目根目录时会导致模型加载失败或重复下载。

## 修复清单

### 修复 1：`start_server.py` — 环境变量改为绝对路径

**文件**: `start_server.py`

**问题**: `config.yaml` 中 `environments` 段的路径（如 `models/pre_trained`）是相对路径。`start_server.py` 直接 `os.environ[key] = str(value)` 不做转换，依赖 CWD 为项目根目录。

**修复**: 设置环境变量前检查路径是否为相对路径，若是则拼接 `project_root` 转为绝对路径：

```python
# 将相对路径转为绝对路径（基于项目根目录）
if not os.path.isabs(abs_value) and not abs_value.startswith('/'):
    abs_value = os.path.join(project_root, abs_value)
    abs_value = os.path.normpath(abs_value)
```

影响变量：`DINOMALY_ENCODER_DIR`、`PRETRAINED_MODELS_DIR`、`TORCH_HOME`、`HF_HOME`、`TRANSFORMERS_CACHE`、`HUGGINGFACE_HUB_CACHE`

---

### 修复 2：`musc_adapter.py` — pretrained_models_dir 绝对路径化

**文件**: `algorithms/musc_adapter.py`

**问题**: `pretrained_models_dir` 默认值 `'models/pre_trained'`（配置读取也是相对路径），直接传给 `open_clip.create_model_and_transforms(cache_dir=...)`。`open_clip` 内部 `download_pretrained_from_url()` 在 `cache_dir` 为 None 时会回退到 `~/.cache/clip`。

**修复**: 在配置加载后，将 `pretrained_models_dir` 转为绝对路径：

```python
if not os.path.isabs(pretrained_models_dir):
    project_root = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
    pretrained_models_dir = os.path.normpath(os.path.join(project_root, pretrained_models_dir))
```

---

### 修复 3：EfficientAD `lightning_model.py` — teacher 权重路径绝对化

**文件**: `algorithms/Anomalib/models/image/efficient_ad/lightning_model.py`

**问题**: `prepare_pretrained_model()` 中 `pretrained_models_dir = Path("./pre_trained/")` 是 CWD 相对路径。若 CWD 不是项目根目录则找不到权重，触发从 GitHub 重新下载。

**修复**: 改用项目根目录绝对路径：

```python
current_file = os.path.abspath(__file__)
project_root = current_file
for _ in range(6):
    project_root = os.path.dirname(project_root)
pretrained_models_dir = Path(project_root) / "models" / "pre_trained"
```

同时补充了缺失的 `import os`。

## 未修复的低风险项

| 路径 | 原因 |
|------|------|
| `DictAS/train.py` 中 `--pretrained_path` 默认值 `./pretrained_weight/...` | DictAS 不是主检测算法，该路径仅在独立训练脚本中使用，通过 API 调用时有适配器传入正确路径 |
| `open_clip/pretrained.py` 中 `~/.cache/clip` fallback | 第三方库代码，不修改。调用链已在修复 2 中确保 `cache_dir` 始终传入 |

- `musc_adapter.py` — MuSc 零样本检测的 CLIP 模型缓存始终写入项目目录
