# 任务管理模块支持在线检测任务改造

**时间**: 2026-06-20 17:25
**作者**: zhouchenghao
**类型**: 功能改造

---

## 背景

任务管理 Tab 原本仅显示离线检测（手动上传/批量检测）的任务记录，页面顶部有提示说明"本页面仅显示离线检测任务"。在线检测（客户端监控）的结果只能在"在线检测" Tab 中查看，无法在统一的任务管理界面中浏览、搜索和管理。

## 核心差异

| 维度 | 离线检测任务 | 在线检测结果 |
|------|-------------|-------------|
| 数据源 | TaskManager.tasks (Dict[str, Task]) | ClientDetectionService.detection_results (deque[dict]) |
| 粒度 | 一次上传 = 一个任务，含多个文件 | 一个文件 = 一个结果条目 |
| 状态 | pending/running/completed/failed/cancelled | 始终为 completed |
| 操作 | 查看/取消/重试/删除 | 仅查看/删除 |

## 改造方案

后端新增 API + 前端合并渲染

### 后端改动

**1. `backend/core/client_monitor_service.py`**
- `__init__`: 新增 `self._result_id_counter: int = 0` 自增计数器，用于唯一定位每条结果
- 新增 `_next_result_id()` 方法: 生成下一个自增ID
- 新增 `get_results(limit, offset)` 方法: 对 detection_results deque 做分页，按时间倒序返回
- 新增 `get_results_count()` 方法: 返回结果总数
- 新增 `delete_result(result_id)` 方法: 按 result_id 删除单条结果
- 在 `_handle_no_match()` 和 `_process_all_detection_results()` 的结果字典中新增 `result_id` 和 `algorithm` 字段

**2. `backend/api/client_monitor.py`
- 新增 `GET /api/client/results?limit=50&offset=0` → 返回 `{total, results, limit, offset}`
- 新增 `GET /api/client/stats` → 返回 `{total, anomaly_count}`
- 新增 `DELETE /api/client/results/{result_id}` → 按 ID 删除单条结果

### 前端改动

**3. `frontend/index.html`
- 移除第 795-798 行的蓝色提示框："本页面仅显示离线检测任务..."
- 筛选栏增加 `<select id="filterType">`：全部类型 / 离线任务 / 在线检测
- 表格增加 "类型" 和 "客户端" 两列，colspan 从 8 改为 10

**4. `frontend/js/task-manager.js`
- 新增 `currentTypeFilter` 全局状态 ('all'/'offline'/'online')
- 新增 `normalizeOnlineTask(r)` 将在线结果标准化为类 Task 格式
- 新增 `loadOnlineResults()` 调用 `/api/client/results` 获取在线检测结果
- 新增 `loadOnlineStats()` 调用 `/api/client/stats` 获取在线检测统计
- 新增 `renderOnlineTaskRow(t)` 渲染在线结果表格行（蓝色"在线"徽章，状态显示"已完成"，无进度/耗时，操作仅"详情"/"删除"）
- 新增 `showOnlineDetailModal(resultId)` / `renderOnlineDetailHtml(result)` 显示在线结果详情（含客户端名、算法、文件名、异常分数、匹配信息、频谱图）
- 修改 `loadTaskList()`: 根据类型筛选，并发加载离线任务和在线结果，合并按时间排序渲染
- 修改 `loadTaskStats()`: 同时加载离线 + 在线统计，合并显示
- 修改 `deleteSingleTask()`: 根据类型调用不同 API 端点
- 修改 `applyFilters()`: 纳入类型筛选重置逻辑
- 修改 `getTaskDuration()`: 在线结果返回 "—"
- 修改 `populateAlgorithmFilter()`: 从全部数据中提取（而非仅离线）

## 关键文件

| 文件 | 改动 |
|------|------|
| backend/core/client_monitor_service.py | 新增 4 方法，2 处结果创建加字段 |
| backend/api/client_monitor.py | 新增 3 个 API 端点 |
| frontend/index.html | 移除提示框，增加类型筛选/列 |
| frontend/js/task-manager.js | 全面重构为双源合并渲染 |

## 验证方法

1. 启动后端服务，前端切换到"任务管理"Tab
2. 确认不再显示"仅显示离线检测任务"提示
3. 统计卡片应同时统计在线检测记录数
4. 类型筛选（全部/离线/在线）能正确过滤
5. 在线结果行显示：蓝色"在线"徽章、"已完成"状态、客户端名、算法、文件数 1、进度"—"
6. 点击在线结果"详情"，弹窗显示文件名、异常分数、客户端名、匹配信息
7. 在线结果行只有"详情"和"删除"按钮，无"取消"/"重试"
