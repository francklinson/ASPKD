# 参考音频列表支持试听功能

**时间**: 2026-06-20 17:45
**作者**: zhouchenghao
**类型**: 功能增强

---

## 背景

参考音频库管理页面只显示 ID、名称、指纹哈希数 等基本信息，用户无法直接试听参考音频的内容来判断该音频是否为想要的参考音频。

## 改造方案

后端增加音频服务端点 + 前端增加试听按钮和播放器

### 后端改动

**1. `backend/api/reference_audio.py`** — 新增音频播放端点
- 新增 `GET /api/reference-audio/audio/{music_id}` 端点
- 通过 fingerprinter 查询参考音频的路径
- 使用 `FileResponse` 返回音频文件
- 根据扩展名自动识别媒体类型 (wav/mp3/flac/ogg 等)
- 路径或文件不存在时返回 404

**2. `backend/api/reference_audio.py`** — 上传流程保存音频文件
- 原有逻辑: 上传后只生成指纹，临时文件被删除，`music_path` 指向不存在的 `data/ref/filename`
- 修复: 在生成指纹后、删除临时文件前，执行 `shutil.copy2(temp_file_path, file_path)` 将音频保存到 `data/ref/` 目录
- 保存失败时仅记录 WARNING 日志，不影响上传主流程

### 前端改动

**3. `frontend/index.html`**
- 参考音频表格表头增加"试听"列（在"指纹哈希数"和"操作"之间）

**4. `frontend/js/reference-audio.js`**
- `renderReferenceList()`: 每行增加试听按钮，绿色播放样式
- 新增 `toggleRefAudioPlay(musicId, buttonElement)`: 独立的音频播放函数
  - 使用独立的 `refAudioPlayer` / `refPlayingButton` 变量，与离线检测音频播放互不干扰
  - 点击播放 → 暂停切换
  - 播放结束或出错时自动重置按钮状态
- 调整 loading/error 状态的 `colspan` 从 4 改为 5

## 数据流

```
用户点击"▶ 试听"按钮
  → toggleRefAudioPlay(music_id, buttonElement)
  → new Audio(`${API_BASE}/api/reference-audio/audio/${music_id}`)
  → 后端 GET /api/reference-audio/audio/{music_id}
    → fingerprinter.get_all_references() 查找路径
    → FileResponse(path=audio_path)
  → 浏览器播放
```
