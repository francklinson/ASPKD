# 数据集构建模块优化

## 变更内容

对应需求文档 `records/docs/需求设计.md` 底部 9 项改进要求。

### 需求 (1) 上传音频增强
- **后端**: `dataset_builder_v2.py` 添加去重保护（`import_history` 字段）
- **后端**: 新增 `POST /api/dataset/import-to-session` 端点，支持将数据追加到已有会话
- **前端**: 离线/在线检测导入时检查是否已导入（去重提示）
- **前端**: 导入时提供"追加到已有会话"选项（合并数据源）
- **前端**: 已导入的任务/结果显示 ✅ 标记并禁用

### 需求 (2) 会话详细来源信息
- **后端**: `DatasetBuildSession` 模型新增 `source_details` 字段
- **后端**: 记录每次导入的来源类型、任务ID、文件名、时间戳
- **前端**: 会话列表项显示来源详情链（如 `📋 离线 #abc12345 → 📤 手动`）

### 需求 (3) 上传和会话控件同一行
- **前端**: 新增 `.mid-row` flex 容器，将上传卡片和会话管理卡片并排显示

### 需求 (4) 不显示 V1/V2 标签
- **前端**: 删除所有 V1/V2 用户可见标签（"V2 会话式构建"描述删除）
- **前端**: "现有数据集（V1）" → "已有数据集"
- 保留 JS 内部的函数名和注释中的 V1 引用（不影响用户体验）

### 需求 (5) 数据统计独立显示
- **前端**: 新增 `.stats-banner` 横向统计横幅，置于页面顶部
- 始终可见，包含 4 项统计数字（总音频、类别、训练集、测试集）和 Chart.js 饼图
- 从原有 V1 折叠区移出

### 需求 (6) 重命名 + 自动加载
- **前端**: "音频试听与标注" → "音频标注"
- **前端**: 点击会话列表项自动调用 `previewV2Session` + `selectSegment`，直接进入标注模式

### 需求 (7) 保留标注信息
- **前端**: 片段列表中显示标注时间和标注者（`annotated_at` 字段）
- 后端原有标注逻辑不变（`manual_label`、`annotated_by`、`annotated_at`）

### 需求 (8) 划分配置功能
- **后端**: 新增 `POST /api/dataset/session/{session_id}/split-preview` 预览划分
- **后端**: 新增 `POST /api/dataset/session/{session_id}/split-and-import` 按配置导入
- **后端**: `confirm_to_dataset` 增强：支持 `reference_audios` 和 `train_ratio` 参数
- **后端**: 划分完成后输出详细的结构化日志（按参考音频分组的 train/test/anomaly 统计）
- **前端**: 点击"确认导入正式数据集"时弹出配置对话框
  - 输入训练:测试比例（默认 10:1）
  - 选择参考音频（全部或指定部分）
  - 预览后确认导入
  - 划分详情在操作日志中显示

### 需求 (9) 紧凑美观布局
- **CSS**: 减少内边距（padding 16→10）、卡片 padding 14→12
- **CSS**: 新增 `.stats-banner` 渐变横条样式
- **CSS**: 新增 `.mid-row` / `.content-row` 弹性布局
- 保留响应式设计，移动端自动切换为纵向排列

## 涉及文件
