# 模型文件与模块映射分析

> 分析时间：2026-06-24
> 项目：ASD for SPK

## 概述

对 `models/pre_trained/` 目录下所有模型文件进行溯源分析，明确每个文件的用途、使用模块和加载方式。

---

## 一、DINOv2 骨干模型（共享基础 backbone）

这些是 Meta 开源的 DINOv2 预训练视觉编码器权重，被多个检测算法共享作为特征提取器。

| 文件 | 大小 | 使用者 | 加载方式 |
|------|------|--------|---------|
| `dinov2_vits14_pretrain.pth` | 85M | **Dinomaly** (dinov2_small 编码器)、**MuSc** (musc_dinov2_b14_\*) | `algorithms/Dinomaly/models/vit_encoder.py` 直接加载 |
| `dinov2_vitb14_pretrain.pth` | 331M | **Dinomaly** (dinov2_base 编码器)、**MuSc** (musc_dinov2_b14_\*) | 同上 |
| `dinov2_vitl14_pretrain.pth` | 1.2G | **Dinomaly** (dinov2_large 编码器)、**MuSc** (musc_dinov2_l14_\*) | 同上 |
| `dinov2_vits14_reg4_pretrain.pth` | 85M | **SubspaceAD** (small_672) | HuggingFace `facebook/dinov2-with-registers-small` → 缓存到 `dinov2-with-registers-small/` |
| `dinov2_vitb14_reg4_pretrain.pth` | 331M | **SubspaceAD** (base_672/518) | HuggingFace `facebook/dinov2-with-registers-base` |
| `dinov2_vitl14_reg4_pretrain.pth` | 1.2G | **SubspaceAD** (large_672/518/336) | HuggingFace `facebook/dinov2-with-registers-large` |

**说明**：
- 前三者不带 registers，供 Dinomaly 推理和 MuSc 零样本检测使用。
- 后三者带 registers，是 SubspaceAD 少样本检测的 backbone。加载时优先使用本地 `.pth` 文件，若缺失则 HuggingFace 自动下载。
- 带 registers 的模型同时被缓存到 `dinov2-with-registers-*/` 子目录（safetensors 格式），缓存目录由 HuggingFace `transformers` API 管理。

## 二、DINOv3 骨干模型（Dinomaly 训练与推理用）

Meta 开源的 DINOv3 视觉编码器。不同于 DINOv2 的直接加载方式，DINOv3 通过 `torch.hub.load_state_dict_from_url()` 下载并缓存到 `hub/` 目录。

| 文件 | 大小 | 使用者 | 加载方式 |
|------|------|--------|---------|
| `dinov3_vits16_pretrain_lvd1689m-08c60483.pth` | 83M | **Dinomaly DINOv3 Small** 编码器 | `dinov3/hub/backbones.py` |
| `dinov3_vits16plus_pretrain_lvd1689m-4057cbaa.pth` | 110M | **Dinomaly DINOv3 Small** (vits16plus) | 同上 |
| `dinov3_vitb16_pretrain_lvd1689m-73cec8be.pth` | 327M | **Dinomaly DINOv3 Base** 编码器 | 同上 |
| `dinov3_vitl16_pretrain_lvd1689m-8aa4cbdd.pth` | 1.2G | **Dinomaly DINOv3 Large** 编码器 | 同上，另被 `segmentors.py`/`detectors.py`/`depthers.py` 引用 |

**文件命名格式**：`dinov3_{arch}_pretrain_{hash}.pth`

## 三、自训练 Dinomaly 模型（软链接 → saved_results）

这两个是实际在 SPK 数据集上训练产出的完整检测器权重（编码器 + 瓶颈 + 解码器）。

```bash
models/pre_trained/dinomaly_dinov2_small.pth
  → algorithms/Dinomaly/saved_results/dinomaly_dinov2_small_dk_qzgy_epoch_1000_....pth

models/pre_trained/dinomaly_dinov3_small.pth
  → algorithms/Dinomaly/saved_results/dinomaly_dinov3_small_dk_qzgy_epoch_1000_....pth
```

**推理流程**：`dinomaly_inference.py` 先根据配置加载对应的 DINOv2/v3 骨干网络编码器，再 `torch.load()` 软链接指向的完整模型文件覆盖全部参数。

**更新方式**：训练新模型后，只需更新软链接指向新文件，无需修改配置。

## 四、CLIP 模型（多模块共享）

OpenAI 的 CLIP 权重，被四个算法模块共同使用。

| 文件 | 大小 | 使用者 |
|------|------|--------|
| `ViT-B-32.pt` | 338M | **MuSc** (musc_clip_b32_512)、**MultiADS**、**DictAS**、**Anomalib AI-VAD** |
| `ViT-B-16.pt` | 335M | **MuSc** (musc_clip_b16_512)、**MultiADS**、**DictAS**、**Anomalib AI-VAD** |
| `ViT-L-14.pt` | 890M | **MuSc** (musc_clip_l14_518)、**MultiADS**、**DictAS**、**Anomalib AI-VAD** |
| `ViT-L-14-336px.pt` | 891M | **MuSc** (musc_clip_l14_336，推荐变体)、**DictAS**（训练/推理主力）、其余同上 |
| `vit_b_16_plus_240-laion400m_e31-8fb26589.pt` | 795M | **DictAS**（OpenCLIP LAION400M 权重）、**MultiADS**、**MuSc** |

**加载路径**：
- `algorithms/MuSc/models/backbone/open_clip/pretrained.py`
- `algorithms/DictAS/open_clip_local/pretrained.py`
- `algorithms/MultiADS/open_clip/pretrained.py`
- `algorithms/Anomalib/models/video/ai_vad/clip/clip.py`

**说明**：`ViT-L-14-336px.pt` 是使用最广泛的 CLIP 模型。DictAS 的训练脚本 `train.py`、`test.py`、`train.sh`、`test.sh` 均硬编码引用此文件。

| 文件 | 大小 | 使用者 |

## 七、自动缓存目录

以下目录不是由本项目直接下载，而是各 AI 框架自动管理的缓存。

| 目录 | 大小 | 内容 | 管理方式 |
|------|------|------|---------|
| `hub/` | 1.5G | DINOv3 系列模型、torchvision 模型 | `torch.hub.load_state_dict_from_url()` 自动下载到 `$TORCH_HOME/hub/` |
| `huggingface/` | 1.9G | `models--facebook--dinov2-with-registers-large` 等 | HuggingFace `transformers` 缓存，`from_pretrained()` 自动管理 |
| `dinov2-{base,large,small}/` | 3.1G | DINOv2 config.json + model.safetensors | `vit_encoder.py` 中 HuggingFace `AutoModel` 缓存 |
| `dinov2-with-registers-{base,large,small}/` | 1.6G | DINOv2+reg config.json + model.safetensors | SubspaceAD 通过 HuggingFace 下载 |
| `resnet18-f37072fd/` | 45M | ResNet-18 模块缓存 | `timm` 或 HuggingFace 缓存 |
| `timm/` | 4K | 空 | `timm.create_model()` 缓存目录 |
| `xet/` | 368K | Git LFS 哈希缓存 | Git LFS 内部使用 |

## 八、磁盘存在但不在 config.yaml 中的文件

以下文件存在于磁盘但未在 `backend/config/config.yaml` 的 `models` 段注册，它们是框架自动缓存：

- `hub/` 目录下的 DINOv3 权重（由 `torch.hub` 管理）
- `huggingface/` 目录下的 HuggingFace 缓存
- `dinov2-*/` 和 `dinov2-with-registers-*/` 目录下的 safetensors 缓存

这些缓存与 `models/pre_trained/` 下的 `.pth` 文件是同一模型的不同格式（PyTorch 权重 vs safetensors），不存在功能缺失风险。

## 九、启动脚本检查覆盖情况

`start_server.sh` 的 `check_models()` 函数检查了全部 29 个 config 注册的模型文件：

- **required_models**（8 个，缺失阻止启动）：DINOv2 ×3 + DINOv2 reg ×3 + ResNet ×2
- **algorithm_models**（19 个，缺失仅警告）：Dinomaly ×2 + ADer ×7 + Anomalib ×3 + BaseASD ×3 + CLIP ×4
- **subdir_models**（2 个，缺失仅警告）：EfficientAD teacher small/medium

覆盖率为 **100%**。
