# 数据集构建页面：批量上传 + 路径映射 + V2 会话集成 + V1 标记

## 背景

2026-06-21 连续修复了数据集构建页面的多个问题，涉及手动上传失败、批量上传支持、路径映射错误、V2 会话集成四个阶段。

---

## 阶段一：手动上传失败修复

### 问题

前端 `formData.append('file', file)` 字段名为 `file`，但 V1 `upload-and-split` 后端期望 `files: List[UploadFile]`（FastAPI 按参数名匹配字段名），导致 422 Validation Error。

### 修复

前端表单字段名统一为 `files`，匹配后端两个端点（`upload-manual`、`upload-and-split`）。

---

## 阶段二：批量上传支持

### 问题

- V1 `upload-manual` 端点只接受单文件 `file: UploadFile`
- 前端文件 `<input>` 没有 `multiple` 属性
- 拖放/选择事件只处理 `files[0]`

### 修复

**后端 `dataset_builder.py`**:
- `upload_manual_audio()` 参数 `file: UploadFile` → `files: List[UploadFile]`
- 循环处理所有上传文件，返回全部 segment_info

**前端 `dataset.html`**:
| 修改 | 说明 |
|------|------|
| 文件 input 添加 `multiple` | 支持 Ctrl/Shift 多选 |
| dropzone 提示 → "支持批量"/"可多选" | UI 提示 |
| drop/change 事件 → `Array.from(files)` | 传递全部文件 |
| `handleFileUpload(file)` → `handleFileUpload(files)` | 接受文件数组 |
| 表单字段名 → `files` | 匹配后端 |
| 增加 HTTP 非 200 响应处理 | 错误信息更清晰 |
| 进度提示显示文件总数 | 如"正在上传 5 个文件..." |
| 上传后重置 file input | 允许重复选择同一文件 |

## 阶段三：路径映射修复（时频图/波形图不显示）

### 问题

2026-06-20 数据目录重组后 `uploads/` 移至 `data/uploads/`，但：
- `get_url_path()` 将 `data/uploads/...` 映射为 `/data/uploads/...`（多一层 `data`）
- 静态挂载点是 `/uploads` → `data/uploads`，URL 无法访问
- `get_file_path()` 将 `/uploads/...` 映射回 `PROJECT_ROOT/uploads/`（缺一层 `data/`）
- V2 各端点直接存储绝对路径而非 URL 路径

### 修复

**`dataset_builder.py`**:
- 新增 `_static_mount_url()` 辅助函数
- `get_url_path()` 重写为按优先级检查已知静态挂载目录：
  `data/spk` → `/data/spk/`、`data/dataset_builder` → `/data/dataset-builder/`、
  `data/uploads` → `/uploads/`、`data/output` → `/output/`
- `get_file_path()` 重写为基于映射表的路由

**`dataset_builder_v2.py`**:
- 导入 `get_url_path` 并应用到 `process_detection_results()`、
  `process_client_detection_results()`、`slice_audio_file()` 的 `file_path`/`spectrogram_path`

### 验证结果

| 磁盘路径 | 映射 URL | HTTP |
|---------|---------|------|
| `data/uploads/dataset_temp/x.wav` | `/uploads/dataset_temp/x.wav` | 200 |
| `data/spk/cat/train/good/x.wav` | `/data/spk/cat/train/good/x.wav` | 200 |
| `data/dataset_builder/sess/x.wav` | `/data/dataset-builder/sess/x.wav` | 200 |
| `data/output/vis/x.png` | `/output/vis/x.png` | 200 |

## 阶段四：V2 会话集成（手动上传自动创建会话）

### 问题

手动上传走 V1 端点，不创建 V2 会话，导致上传的数据无法被「一键构建数据集」识别。

### 修复

**后端 `dataset_builder_v2.py`**:
- `from-manual` 端点新增 `skip_slicing: bool = Form(False)` 参数
- `process_manual_upload()` 新增 `skip_slicing` 分支：
  - `skip_slicing=true`：跳过 Shazam 切分，每个上传文件作为一整段保存，生成时频图
  - `skip_slicing=false`：原有 Shazam 匹配切分逻辑

**前端 `dataset.html`**:
- `handleFileUpload` 改为调用 V2 `from-manual` 端点（不再调 V1 `upload-manual`/`upload-and-split`）
- 手动模式：`skip_slicing=true`
- 自动模式：`skip_slicing=false`，同时传 `auto_predict=false`
- 上传成功后调用 `loadV2Sessions()` 刷新右侧会话列表

### 用户操作流

```
手动上传模式：
  上传文件 → V2 from-manual(skip_slicing=true)
  → 创建 V2 会话 + 文件整段保留
  → 右侧「数据集构建会话」列表出现新会话
  → 预览 → 标注 → 「一键构建数据集」

自动切分模式：
  上传文件 → V2 from-manual(skip_slicing=false)
  → 创建 V2 会话 + Shazam 匹配切分
  → 右侧「数据集构建会话」列表出现新会话
  → 预览 → 标注 → 「一键构建数据集」
```
