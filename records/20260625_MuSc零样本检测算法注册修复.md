# MuSc 零样本检测算法注册修复

> 修复时间：2026-06-25

## 问题描述

执行零样本检测报错：

```
获取结果失败: 未找到算法: musc_clip_l14_336。
可用算法: ['dinomaly_dinov3_small', ..., 'musc', 'subspacead_dinov2_small_672']
```

`musc_clip_l14_336` 不在可用算法列表中。虽然 `config.yaml` 和 `zero_shot.py` 的 `BACKBONES` 字典中均有定义，但 `AlgorithmRegistry` 中只注册了 `musc`（来自 `other_adapters.py` 的桩适配器），8 个完整 MuSc 变体（`musc_clip_b32_512`、`musc_clip_l14_336` 等）全部缺失。

## 根因

`algorithms/musc_adapter.py` 缺少 `import torch`。模块顶层有 3 处方法签名使用了 `torch.Tensor` 类型注解（第 178、218、293 行），Python 在类定义阶段就会求值这些注解，但 `torch` 未导入，抛出 `NameError: name 'torch' is not defined`。

`factory.py` 的 `_import_adapters()` 用 `try/except` 包裹每个适配器的导入：

```python
try:
    from . import musc_adapter
except Exception as e:
    print(f"[algorithms] musc_adapter 导入失败: {e}")
```

该异常被静默捕获，导致 `musc_adapter.py` 中 8 个 `@register_algorithm("musc_*")` 装饰器全部未执行，因此 `AlgorithmRegistry` 中没有这些算法的条目。

## 修复清单

### 修复 1：`musc_adapter.py` — 添加缺失的 `torch` 导入

**文件**: `algorithms/musc_adapter.py`

**问题**: 缺少 `import torch`，类型注解 `torch.Tensor` 在模块导入时报 `NameError`。同时 `F.interpolate()`（第 280 行）也缺少 `import torch.nn.functional as F`。

**修复**: 在导入段末尾添加：

```python
import torch
import torch.nn.functional as F
```

注：该修复在提交 `71614b9` 中已提前合入。

### 修复 2：`factory.py` — `list_available_algorithms()` 不触发适配器导入

**文件**: `algorithms/factory.py`

**问题**: `list_available_algorithms()` 仅返回 `AlgorithmRegistry.list_algorithms()`，不调用 `_import_adapters()`。若在 `create_detector()` 之前调用，返回空或不完整列表。

**修复**: 在 `list_available_algorithms()` 中添加 `_import_adapters()` 调用。

```python
def list_available_algorithms() -> List[str]:
    _import_adapters()
    return AlgorithmRegistry.list_algorithms()
```

## 验证结果

- `create_detector("musc_clip_l14_336", ...)` 成功创建 `MuScCLIP_L14_336` 实例
- `AlgorithmRegistry` 中包含全部 8 个 MuSc 变体
- `list_available_algorithms()` 返回完整算法列表
