# 多项 Bug 修复与布局调整

## 概述
本次会话集中修复了离线检测、数据集构建和页面布局的多个问题，涉及前后端共 14 个文件。

---

## 修复列表

### 1. `name 'ref_file' is not defined` 运行时错误

- **文件**: `backend/core/task_manager.py`
- **问题**: 重构 `ref_file=None` 硬编码后，`_execute_task` 方法中遗留了 3 处对未定义变量 `ref_file` 的引用（第 400、401、505 行）
- **修复**: 
  - 删除第 400-403 行的 `if ref_file:` / `else:` 死分支
  - 第 505 行引用改为 `Shazam 自动匹配` 文字说明

### 2. 离线检测结果图片无法显示（数据目录重组遗留）

- **根因**: 提交 `1ce0941`（`output/ → data/output/`）更新了部分后端文件，但遗漏了算法模块和多个 API 文件
- **影响**: 算法将图片保存到 `output/vis/`，但 FastAPI 静态挂载点从 `data/output/vis/` 提供服务 → 404
- **修复涉及 8 个文件**:

| 文件 | 修改内容 | 处数 |
|------|---------|------|
| `algorithms/Dinomaly/utils.py` | `'output', 'vis'` → `'data', 'output', 'vis'` | 5 |
| `algorithms/Dinomaly/dinomaly_inference.py` | `"output/vis"` → `"data/output/vis"` | 2 |
| `backend/api/few_shot.py` | `"output", "vis"` → `"data", "output", "vis"` | 3 |
| `backend/api/zero_shot.py` | 同上 | 3 |
| `backend/api/detection.py` | `"output", "exports"` → `"data", "output", "exports"` | 2 |
| `backend/api/client_monitor.py` | 同上 | 2 |
| `backend/api/local_monitor.py` | 同上 | 2 |
| `backend/core/task_manager.py` | `"uploads"` → `"data", "uploads"` | 1 |
| `frontend/js/task-manager.js` | 更新清理提示消息中的路径 | 1 |

- **文件迁移**: `output/vis/` 中 27 个历史图片 → `data/output/vis/`，`uploads/` 中 2 个任务 → `data/uploads/`

### 3. Shazam 检测阈值偏低导致切分不准

- **文件**: `tools/preprocessing.py`
- **问题**: 
  - `min_match_ratio=0.0` 完全禁用匹配率过滤，虚假匹配（匹配率仅 0.27%）被当作有效片段
  - 不可靠片段（`is_reliable=False`）未在 `_process_audio_serial` 中被过滤
- **修复**:
  - 删除 `min_match_ratio=0.0` 覆盖，恢复使用 `SegmentLocatorConfig` 默认值 `0.01`（1%）
  - 新增不可靠片段过滤，跳过置信度 < `threshold × 2` 的低质量匹配

### 4. 离线检测结果未进入数据集构建成为预标注数据

- **前端文件**: `frontend/dataset.html`
- **问题**: `previewV2Session` 函数中残留 17 行死代码（第 3646-3662 行），引用了已出作用域的变量 `s`，导致 `ReferenceError`，预览会话时静默失败
- **修复**: 删除死代码

- **前端文件**: `frontend/js/offline-detection.js`
- **问题**: 历史结果的 `audio_slice_path` 重建路径为旧格式 `slice/audio/xxx.wav`，应为 `data/output/slices/audio/xxx.wav`
- **修复**: 将重建路径格式更新为 `data/output/slices/audio/xxx.wav`

### 5. 数据集构建页面布局调整

- **文件**: `frontend/dataset.html`
- **问题**: V1 统计卡片挤在顶栏与 V2 上传卡片争空间；V1 主体（播放器+片段列表）插在 V2 上传和 V2 会话之间，页面视觉混乱
- **修复**:
  - 统计卡片从顶栏移出，放入可折叠的 "现有数据集（V1）" 区域
  - V2 会话面板移到 V1 区域上方，V2 内容集中排列
  - 新增 `toggleV1Section()` 折叠函数，V1 区域默认折叠
  - 初始化时临时展开 V1 以便 Chart.js 正确渲染，渲染后自动折叠

### 6. 设备列表加载报错

- **文件**: `frontend/js/app.js`
- **问题**: `loadAvailableDevices()` 中 `#deviceInfo` 等元素在离线检测 Tab 下不存在，访问 `.textContent` 时抛 `TypeError`
- **修复**: 增加 `deviceSelect`/`deviceInfo`/`monitorDeviceSelect`/`monitorDeviceInfo` 的 null 安全检查
