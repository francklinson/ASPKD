# 数据集构建功能完善记录

**日期**: 2026-06-20

## 改造目标

完善数据集构建功能，统一支持三种数据来源：
1. **手动上传** — 用户上传音频文件，Shazam 匹配切分
2. **离线检测** — 从已完成的离线检测任务中提取结果
3. **在线/客户端检测** — 从客户端检测结果中提取数据

实现完整的"选择来源 → 创建会话 → 预览片段 → 标注 → 确认导入"流程。

## 数据集存储设计

```
data/spk/{category_name}/
  ├── train/good/         # 训练集-正常样本 (.wav + .png)
  ├── test/good/          # 测试集-正常样本
  └── test/anomaly/       # 测试集-异常样本
```

划分策略：正常样本按 10:1 分配 train:test，异常样本强制进 test/anomaly。

## 修改文件详情

### 1. backend/api/dataset_builder_v2.py

**新增功能**：

a) **会话持久化** (`_load_sessions_from_disk`)
   - 模块加载时扫描 `data/dataset_builder/*.json`
   - 反序列化为 `DatasetBuildSession` 对象恢复到内存
   - 确保服务重启后会话不丢失

b) **POST /from-client-detection** — 从在线检测结果创建会话
   - 请求体: `{ result_ids: List[int], auto_annotate_threshold: float }`
   - 处理函数 `process_client_detection_results()`
   - 从 `client_detection_service.detection_results` 按 ID 获取结果
   - 自动提取音频切片路径（支持直接路径和从 overlay_path 推断）
   - 保留模型预测分数作为预标注
   - 高置信度（>threshold）自动标注

c) **辅助函数 `_find_audio_slice_for_client_result()`**
   - 尝试多种方式定位客户端检测结果的音频切片：
     1. 直接取 `audio_slice_path`
     2. 从 `overlay_path` 反向推断文件名
     3. 搜索可能的切片目录

d) **GET /available-tasks** — 获取可用于数据集构建的已完成离线任务
e) **GET /available-client-results** — 获取可用于数据集构建的在线检测结果

### 2. backend/main.py

- 新增 `dataset_builder_v2` 导入和路由注册
- 新增静态文件挂载 `/data/dataset-builder` → `data/dataset_builder/`

### 3. frontend/dataset.html

**新增 UI 元素**：

a) **数据来源 Tab**（上传卡片顶部）
   - 手动上传 / 离线检测 / 在线检测 三个 Tab
   - 点击切换显示对应面板

b) **离线检测面板**
   - 调用 `GET /api/dataset/available-tasks` 加载已完成任务
   - 任务列表显示（ID、算法、文件数、结果数）
   - 选中后点击"导入到数据集构建"

c) **在线检测面板**
   - 调用 `GET /api/dataset/available-client-results` 加载结果
   - 结果列表显示（文件名、客户端、异常分数、状态）
   - 支持多选（复选框），批量导入

d) **数据集构建会话管理面板**
   - 显示所有 V2 构建会话列表
   - 会话状态徽章（已完成/处理中/待处理）
   - 操作按钮：预览、删除
   - 刷新按钮

e) **V2 标注集成**
   - 预览会话时显示片段列表（含预标注分数）
   - 提交标注按钮自动识别 V2 会话，调用 V2 标注端点
   - "确认导入正式数据集"按钮（仅 V2 会话时显示）

**新增 JS 函数**：
- `switchDataSource()` / `toggleSessionPanel()` — UI 切换
- `loadOfflineTasks()` / `loadOnlineResults()` — 数据加载
- `importSelectedTask()` / `importSelectedClientResults()` — 导入流程
- `loadV2Sessions()` / `renderV2SessionItem()` — 会话管理
- `previewV2Session()` / `renderV2SegmentsList()` — 预览
- `confirmV2SessionToDataset()` — 确认导入
- `submitAnnotation()` 重写 — 支持 V1/V2 双端点

## 数据流

```
手动上传 ─────────────────┐
                          ├──→ V2 DatasetBuildSession ──→ 人工标注 ──→ data/spk/
离线检测任务 ──→ from-detection ──→        ↑
在线检测结果 ──→ from-client-detection ──→  (预标注保留)
```

## 验证方法

1. **后端验证**：
   ```bash
   curl http://localhost:8004/api/dataset/available-tasks
   curl http://localhost:8004/api/dataset/available-client-results
   curl -X POST http://localhost:8004/api/dataset/from-client-detection \
     -H "Content-Type: application/json" \
     -d '{"result_ids": [1, 2], "auto_annotate_threshold": 0.8}'
   curl http://localhost:8004/api/dataset/sessions
   ```

2. **前端验证**：
   - 访问 `http://localhost:8004/dataset`
   - 切换数据来源 Tab
   - 从离线/在线检测导入数据
   - 预览会话、标注、确认导入

3. **持久化验证**：
   - 创建会话后重启服务
   - 检查会话是否仍在列表中
