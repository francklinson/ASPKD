### 剩余算法状态标记 (2026-07-12)

以下 33 个算法经评估后**暂不实现训练/推理功能**，原因及决策如下：

#### 设计如此 — 无需训练（16 个）

| 算法 | 原因 |
|------|------|
| padim, dfkde | 特征嵌入类 — 直接对预训练特征做概率建模，天生无需训练 |
| MuSc 全族 (8个) | 零样本检测 — 利用 CLIP/DINOv2 预训练特征做多尺度对比，设计上不训练 |
| SubspaceAD 全族 (6个) | 少样本检测 — PCA 子空间建模，只需几张参考图，设计上不训练 |

> 决策：**永久保持仅推理状态**，不在训练页面列出。

#### 外部依赖缺失 — 不引入（6 个）

| 算法 | 缺失依赖 | 不引入原因 |
|------|---------|-----------|
| denseae, cae, vae, aegan, differnet | TensorFlow + Keras | 与 PyTorch 环境冲突风险，且是早期基础算法（自编码器类），性能已被 Dinomaly/Anomalib 超越 |
| vlm_ad | Ollama 服务 | 需额外部署语言模型服务，增加运维复杂度，非核心需求 |

> 决策：**永久保持不可用状态**，不安装额外依赖。

#### 实现成本过高 — 不值得（6 个）

| 算法 | 原因 |
|------|------|
| efficient_ad | 需 ImageNette 知识蒸馏流程，师生网络交替训练，实现复杂 |
| fastflow, stfpm, uflow, cfm | 训练需下载额外预训练 backbone，离线环境不可用；且已有同范式替代（CFLow-AD 替代 fastflow/uflow，ViTAD 替代 stfpm） |
| hiad, multiads, dictas | 空壳存根，无从零实现的算法论文/代码，且非主流方案 |

> 决策：**永久保持当前状态**，已有更优替代方案。

#### 不适用（2 个）

| 算法 | 原因 |
|------|------|
| audio_feature_cluster | 音频专用，图片检测不适用 |
| musc (旧), subspacead (旧) | 新版独立适配器已覆盖全部功能，旧版入口保留仅为向后兼容 |

> 决策：**永久保持不可用状态**。
