## 目标

设计并实现测试脚本，验证训练页面列出的所有可训练算法都能正常完成训练流程（至少完成 1 个 epoch，无报错退出）。

## 实现方案

### 文件：`tests/test_all_training.py`

创建新的训练测试脚本，核心设计：

1. **直接调用后端训练 API 的逻辑**（不启动 HTTP 服务器），通过 `_run_dinomaly_training`、`_run_anomalib_training` 等函数模拟训练过程，或直接构造训练命令用子进程执行
2. **使用 MVTec AD 数据集的 capsule 类别**（训练样本少，训练快）
3. **最小训练参数**：`total_iters=100`（Dinomaly/Dinomaly2）/ `max_epochs=1`（Anomalib）/ `epoch_full=1`（ADer）
4. **超时设置**：每个算法 600 秒超时
5. **GPU 显存管理**：每个算法测试完清理 GPU 缓存
6. **结果报告**：JSON 格式，记录每个算法的训练状态、耗时、错误信息

### 脚本结构

```python
# tests/test_all_training.py
# 核心结构：
# 1. ALGORITHM_TEST_CASES 定义所有测试用例（family, algorithm_id, 训练参数）
# 2. 每个算法族一个 handler 函数，构造训练命令并执行
# 3. 子进程执行训练，捕获 stdout/stderr
# 4. 通过 returncode 判断成功/失败
# 5. 收集结果生成 JSON 报告
```

### 命令行用法

```bash
# 全量测试
python tests/test_all_training.py

# 指定算法族
python tests/test_all_training.py --families dinomaly ader

# 快速模式（每族选 1 个代表性算法）
python tests/test_all_training.py --quick

# 指定超时
python tests/test_all_training.py --timeout 300

# 指定数据集类别
python tests/test_all_training.py --category capsule
```

### 验证标准

- **通过**：子进程 returncode=0，训练正常完成
- **失败**：子进程 returncode≠0 或超时
- **跳过**：算法标记为不可训练
