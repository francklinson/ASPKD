## 测试方法

编写 Python 测试脚本 `tests/test_all_training.py`，通过 API 接口逐个启动训练任务并检查结果。

每个测试用例：
1. 调用 `POST /api/training/start` 启动训练（total_iters=100, 1 epoch 快速验证）
2. 轮询 `GET /api/training/status/{task_id}` 等待完成
3. 检查最终状态为 `completed` 而非 `failed`

### 关键修改文件
- 新建 `tests/test_all_training.py` — 测试脚本
- 修改 `backend/api/training.py` — 确保 ADer 训练使用 benchmark 配置 + opts 覆盖

### 测试脚本设计
```python
# 伪代码
TEST_CASES = [
    # (family, algorithm_name, model_type, model_size)
    ("dinomaly", "", "dinov3", "small"),
    ("dinomaly", "", "dinov3", "base"),
    ...
    ("anomalib", "patchcore", "", ""),
    ("anomalib", "cfa", "", ""),
    ...
    ("ader", "mambaad", "", ""),
    ("ader", "invad", "", ""),
    ...
]

for family, algo, model_type, model_size in TEST_CASES:
    # 1. POST /api/training/start
    resp = requests.post("/api/training/start", json={
        categories: ["bottle"],
        data_source: "spk",
        algorithm_family: family,
        algorithm_name: algo,
        model_type: model_type,
        model_size: model_size,
        total_iters: 100,  # 最小迭代，快速验证
    })
    task_id = resp.json()["task_id"]
    
    # 2. 轮询等待完成
    while True:
        status = requests.get(f"/api/training/status/{task_id}").json()
        if status["status"] in ("completed", "failed"):
            break
        time.sleep(5)
    
    # 3. 检查结果
    if status["status"] == "completed":
        print(f"✓ {family}/{algo or model_type+model_size}")
    else:
        print(f"✗ {family}/{algo or model_type+model_size}: {status['progress']}")
```

## 验证标准
