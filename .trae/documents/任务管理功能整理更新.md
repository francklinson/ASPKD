# 任务管理功能整理与可用性更新

## 背景

当前任务管理页面（`#tasks`）存在以下问题：
1. **三套独立的任务存储互不相通**：`TaskManager`（离线音频检测）、`CUSTOM_TASKS`（自定义图片检测）、`TRAINING_TASKS`（模型训练）各自维护内存字典，任务管理页面只展示了前两种，训练任务完全不在列表中
2. **任务信息不完整**：离线任务列表缺少 `started_at`/`completed_at` 时间、`algorithm_family` 等关键字段；自定义检测任务不在列表中
3. **统计卡片只反映部分任务**：统计只聚合离线+在线，不包括训练任务和自定义检测任务
4. **筛选不完整**：算法筛选下拉只含离线检测算法，无训练任务/自定义检测的算法族
5. **任务类型覆盖不全**：只有"离线"和"在线"两种类型标签，没有"训练"和"图片检测"类型

## 修改方案

### 1. 后端：新增统一任务列表 API

**文件**: `backend/api/tasks.py`

新增 `GET /api/tasks/all` 端点，聚合四种任务源：
- 离线音频检测（`task_manager.tasks`）
- 在线检测（`client_manager`）
- 自定义图片检测（`CUSTOM_TASKS`）
- 模型训练（`TRAINING_TASKS`）

返回统一格式的任务列表，每条任务包含：
```python
{
    "id": "xxx",
    "type": "offline_audio" | "online" | "custom_detection" | "training",
    "status": "pending" | "running" | "completed" | "failed" | "cancelled",
    "algorithm": "...",
    "algorithm_family": "...",  # dinomaly/anomalib/ader/dinomaly2
    "progress": 0.0-100.0,
    "created_at": "ISO时间",
    "started_at": "ISO时间或null",
    "completed_at": "ISO时间或null",
    "file_count": 0,
    "error": null,
    # 类型特有字段
    "client_name": "...",  # online
    "category": "...",     # training
}
```

### 2. 后端：补全 `list_tasks` 返回字段

**文件**: `backend/core/task_manager.py`

`list_tasks()` 当前只返回 `id/status/algorithm/file_count/progress/created_at`，需补充 `started_at`、`completed_at`、`error` 字段。

### 3. 后端：新增统计聚合 API

**文件**: `backend/api/tasks.py`

新增 `GET /api/tasks/stats/all`，聚合四种任务源的统计数据，按类型分组。

### 4. 前端：更新任务管理页面

**文件**: `frontend/js/task-manager.js`

- `loadTaskList()` 改为调用 `/api/tasks/all`，统一获取所有类型的任务
- 新增任务类型：`训练`（紫色标签）、`图片检测`（橙色标签）
- 统计卡片增加"训练任务"数量显示
- 算法筛选下拉自动从任务列表中提取算法名/算法族
- 类型筛选增加"训练"和"图片检测"选项

**文件**: `frontend/index.html`

- 类型筛选 `<select id="filterType">` 增加选项：`训练任务`、`图片检测`
- 统计卡片区域增加训练任务统计

### 5. 前端：新增训练任务和自定义检测任务的详情展示

**文件**: `frontend/js/task-manager.js`

- `renderTrainingTaskRow()` — 渲染训练任务行（算法族、类别、训练进度、耗时）
- `renderCustomDetectionTaskRow()` — 渲染自定义检测任务行（算法、图片数、异常数）
- `showTrainingDetailModal()` — 训练任务详情弹窗
- `showCustomDetectionDetailModal()` — 自定义检测详情弹窗

## 涉及文件

| 文件 | 修改内容 |
|------|---------|
| `backend/api/tasks.py` | 新增 `/all` 和 `/stats/all` 端点，聚合四类任务 |
| `backend/core/task_manager.py` | `list_tasks()` 补充 `started_at`/`completed_at`/`error` 字段 |
| `frontend/js/task-manager.js` | 重写 `loadTaskList()`，新增行渲染和详情函数 |
| `frontend/index.html` | 筛选栏和统计卡片增加训练/图片检测类型 |

## 验证

1. 启动后端，检查 `/api/tasks/all` 返回包含所有类型的任务
2. 任务管理页面展示训练任务、图片检测任务
3. 统计卡片数字正确
4. 类型/状态/算法筛选正常工作
5. 点击训练任务可查看详情
